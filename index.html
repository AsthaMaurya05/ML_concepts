    What is Machine Learning?

    Machine Learning (ML) is a branch of Artificial Intelligence where a computer learns patterns from data instead of being explicitly programmed.

    Instead of hand-writing a rule like:

    IF Salary > 50,000 → Loan = Approved

    …we hand the machine data, and it learns that relationship on its own.

    Types of Machine Learning

    ML splits into two main categories, depending on whether the training data comes with answers attached.

    Supervised

    Inputs (X) paired with correct outputs (Y). The model learns the mapping between them.

    • House price prediction
    • Spam email detection
    • Disease prediction
    • Student marks prediction

    Goal: predict the correct output for unseen data.

    Algorithms: Linear Regression, Logistic Regression, Decision Tree, Random Forest, KNN, SVM, Naive Bayes.

    Unsupervised

    Only input data — no labels. The model looks for hidden structure on its own.

    • Customer segmentation
    • Fraud pattern discovery
    • Recommendation systems

    Algorithms: K-Means, DBSCAN, Hierarchical Clustering.

    The ML Pipeline

    Nearly every supervised ML project follows the same sequence — only the model in the middle changes.

    Collect Data→ Clean Data→ EDA→ Feature Eng.→ Train/Test Split→ Choose Model→ Train→ Predict→ Evaluate→ Improve

    Only the model changes — Linear Regression, Decision Tree, Random Forest, etc. Everything else in the pipeline stays nearly the same.

    Train–Test Split

    The dataset is divided into two parts so we can measure performance on data the model hasn't seen.

    Training Data

    Used to train the model. Usually 70–80% of the data.

    Testing Data

    Used to evaluate performance on unseen data. Usually 20–30%.

    1000 samples → 800 training / 200 testing

    Regression vs Classification

    Regression

    Output is a continuous value.

    • House price
    • Temperature
    • Stock price

    Algorithm: Linear Regression

    Classification

    Output belongs to a category.

    • Spam / Not spam
    • Cancer / No cancer
    • Pass / Fail

    Algorithms: Logistic Regression, Decision Tree, Random Forest, KNN, SVM, Naive Bayes

    Linear Regression

    A supervised algorithm for predicting continuous values. It finds the best-fit straight line between input and output.

    y = mx + c y → predicted value · m → slope · x → input feature · c → intercept
    Area of House→ Linear Regression→ Predicted Price

    Error Metrics (Regression)

    After prediction, we measure how wrong the model is. Error = Actual − Predicted.

    Mean Absolute Error (MAE)

    MAE = (1/n) · Σ |y − ŷ|

    Pros

    • Easy to understand
    • Less affected by outliers

    Cons

    • Doesn't penalize large errors much

    Mean Squared Error (MSE)

    MSE = (1/n) · Σ (y − ŷ)²

    Pros

    • Penalizes large mistakes

    Cons

    • Unit becomes squared
    • Sensitive to outliers

    Root Mean Squared Error (RMSE)

    RMSE = √MSE

    Pros

    • Same unit as target variable
    • Easier to interpret

    Cons

    • Still sensitive to outliers

    Logistic Regression

    Despite the name, this is used for classification, not regression. It predicts the probability that an input belongs to a class, using the sigmoid function.

    P = 1 / (1 + e^−z) Output is squeezed into the range 0 → 1
    Probability = 0.92→ Cancer

    Confusion Matrix & Classification Metrics

    A confusion matrix compares predicted values against actual values.

    Predicted Pos.Predicted Neg.
    Actual Pos.TPFN
    Actual Neg.FPTN

    FP — Type-I error. e.g. a healthy person predicted sick.

    FN — Type-II error. e.g. a cancer patient predicted healthy.

    Accuracy

    Accuracy = (TP + TN) / (TP + TN + FP + FN)

    Best when classes are balanced.

    Precision

    Precision = TP / (TP + FP)

    Of everything predicted positive, how many actually were? Matters most when false positives are costly — e.g. spam detection.

    Recall

    Recall = TP / (TP + FN)

    Of all actual positives, how many did we catch? Critical in medical diagnosis — missing a disease can be dangerous.

    F1 Score

    F1 = 2 · (Precision · Recall) / (Precision + Recall)

    Harmonic mean of precision and recall — used when both matter equally.

    Decision Tree

    A supervised algorithm that predicts by asking a series of questions.

    Age > 18?→ Income > 50K?→ Loan Approved

    Parts of a tree

    • Root node — the first, most important split.
    • Decision node — intermediate questions.
    • Leaf node — the final prediction, no further splitting.

    Entropy

    Entropy(S) = −Σ pᵢ · log₂(pᵢ) 0 → pure · 1 → completely mixed. Lower is better.

    Information Gain

    Tells us how much uncertainty decreases after a split.

    Information Gain = Entropy(Parent) − Weighted Entropy(Children)

    The split with the highest information gain is chosen next.

    Gini Index

    Gini = 1 − Σ pᵢ²

    Trees typically split using either Gini Index or Entropy.

    Algorithm, step by step

    Calc. Entropy→ Calc. Info Gain→ Pick Best Split→ Create Root→ Repeat per Branch→ Stop when Pure
    Limitation: a single tree can memorize training data, causing overfitting. Fix: Random Forest.

    Random Forest

    An ensemble method that combines many Decision Trees, each trained on a different random sample of the data.

    Classification

    Final prediction by majority vote across trees.

    Regression

    Final prediction by averaging across trees.

    Advantages

    • Higher accuracy
    • Less overfitting
    • More robust

    Disadvantages

    • Slower
    • Harder to interpret

    K-Nearest Neighbors (KNN)

    Predicts based on the nearest data points — usually measured with Euclidean distance.

    Distance = √[(x₂ − x₁)² + (y₂ − y₁)²]
    New Student→ Find 5 Nearest→ Majority Class→ Prediction

    Choosing K

    Small K

    Sensitive to noise.

    Large K

    Smoother predictions, may ignore local patterns.

    Support Vector Machine (SVM)

    A supervised algorithm that finds the best boundary (a hyperplane) separating classes — "best" meaning the one with the widest possible margin to the nearest points of each class.

    Key terms

    • Hyperplane — the decision boundary separating classes.
    • Margin — the gap between the hyperplane and the nearest points of each class. SVM maximizes this.
    • Support vectors — the closest data points to the hyperplane; they alone determine its position.

    Kernel trick

    When data isn't linearly separable in its original space, a kernel function projects it into a higher dimension where a straight hyperplane can separate it.

    Kernels: Linear · Polynomial · RBF (Gaussian) RBF is the most common default for non-linear boundaries

    Advantages

    • Effective in high-dimensional spaces
    • Robust against overfitting, especially with a clear margin

    Disadvantages

    • Slower to train on large datasets
    • Less interpretable, sensitive to kernel/parameter choice

    Naive Bayes

    A probabilistic classifier built on Bayes' Theorem. It's called "naive" because it assumes every feature is independent of the others — rarely true in practice, but the model still performs surprisingly well.

    P(Class | Features) = [P(Features | Class) · P(Class)] / P(Features) Posterior = (Likelihood · Prior) / Evidence

    Worked intuition — spam detection

    For an email containing the words "free" and "offer", the model compares:

    P(Spam) · P("free"|Spam) · P("offer"|Spam) P(Not Spam) · P("free"|Not Spam) · P("offer"|Not Spam)

    Whichever product is larger decides the predicted class.

    Email Text→ Word Probabilities→ Compare Classes→ Spam / Not Spam

    Common variants

    Gaussian NB

    For continuous features, assumed normally distributed.

    Multinomial NB

    For discrete counts — classic choice for text/spam classification.

    Advantages

    • Fast to train and predict
    • Works well with high-dimensional data (e.g. text)
    • Needs relatively little training data

    Disadvantages

    • Independence assumption is rarely realistic
    • Can be outperformed by models that capture feature interactions

    K-Means Clustering

    An unsupervised algorithm — no labels involved. It groups data points into K clusters based purely on similarity (distance), so points inside the same cluster are close together and different clusters are far apart.

    Rings mark the three cluster centroids; points are assigned to whichever centroid is nearest.

    Key terms

    • K — the number of clusters to form, chosen upfront.
    • Centroid — the mean position of all points currently assigned to a cluster.
    • Inertia (WCSS) — within-cluster sum of squares; how tightly packed each cluster is. Lower is better.
    Distance(point, centroid) = √Σ (xᵢ − cᵢ)² Same Euclidean distance idea used in KNN, applied here to centroids instead of neighbors

    Algorithm, step by step

    Pick K→ Init. Centroids→ Assign Points→ Recompute Centroids→ Repeat until Stable

    Steps 3 and 4 repeat until centroids stop moving (convergence) or a maximum number of iterations is reached.

    Choosing K — the Elbow Method

    Plot inertia against different values of K. Inertia keeps dropping as K increases, but the rate of improvement slows sharply at some point — the "elbow" — which is a good candidate for K.

    Advantages

    • Simple and fast on large datasets
    • Easy to interpret and implement

    Disadvantages

    • Must choose K in advance
    • Sensitive to initial centroid placement
    • Assumes roughly round, similarly-sized clusters
    • Sensitive to outliers
    K-Means is the go-to baseline for clustering. When clusters aren't round or K isn't obvious, DBSCAN or Hierarchical Clustering are the usual next options.

    Overfitting, Underfitting & Bias–Variance

    Overfitting

    The model learns training data too well, including noise.

    Train Acc.
    Test Acc.

    Fixes: more data, regularization, Random Forest, pruning, cross-validation.

    Underfitting

    The model is too simple and fails to learn patterns.

    Train Acc.
    Test Acc.

    Fixes: better features, a more complex model, more training.

    Good fit

    Train Acc.
    Test Acc.

    Goal: low training error and low testing error.

    Bias vs Variance

    Bias — error from overly simple assumptions. High bias misses patterns → underfitting.

    Variance — error from being too sensitive to training data. High variance memorizes noise → overfitting.

    ModelBiasVariance
    UnderfittingHighLow
    OverfittingLowHigh
    Good modelModerate/LowModerate/Low
